Feedback & Programme Directory Page — Validation Report

The Feedback & Programme Directory page was validated using the W3C Markup Validation Service. The page passed with no errors. A small number of warnings were present relating to the use of aria-labelledby attributes on section elements, but these do not affect the validity of the HTML and were retained as they improve accessibility for screen reader users. All form input elements were correctly associated with their labels, and all images included appropriate alt attributes. The validation confirmed that the page follows correct HTML5 document structure with a single <h1>, proper heading hierarchy, and valid use of semantic elements throughout.

Reflection on the Validation Process

Validating all three pages using the W3C Markup Validation Service was a valuable and eye-opening part of the development process. It reinforced the importance of writing clean, standards-compliant HTML rather than relying solely on visual appearance in the browser — a page can look correct while containing structural errors that affect accessibility, search engine indexing, and cross-browser compatibility.

One key lesson was the importance of correctly associating form labels with their inputs using matching for and id attributes on the Feedback page. Initially, some label-input pairs had mismatched IDs which the validator flagged. Correcting these not only resolved the validation warnings but also improved the accessibility of the form for keyboard and screen reader users.

Another learning from the validation process was the need to ensure heading hierarchy is strictly maintained. Early drafts of the Content Page had an <h3> appearing before an <h2> in one section, which the validator flagged as a structural issue. Fixing this improved the logical flow of the document and made the page easier to navigate for screen reader users.

Overall, the validation process confirmed that all three pages comply with HTML5 standards and are free from errors. Any warnings encountered were related to ARIA attributes used to enhance accessibility and were intentionally retained.
